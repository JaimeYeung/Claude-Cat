# Cat Gatekeeper — PRD

Date: 2026-05-03

## 背景与目标

使用 Claude Code 时，用户经常离开屏幕或分心，错过 Claude 等待操作的时机，导致会话卡住长达数十分钟。

目标是做一款 macOS 桌面宠物 app，让猫猫常驻桌面，在 Claude Code 需要用户时主动发出可见+可听的提醒，同时支持个性化定制和休息提醒。

---

## 用户故事

**作为用户，我希望：**

1. 打开 app 后，我的猫猫出现在桌面上，一直在那里陪着我
2. 当 Claude Code 在等我操作时，猫猫能立刻提醒我，不需要我盯着终端
3. 猫猫叫我的方式是我自己设定的称呼，有沉浸感
4. 我可以上传自己猫猫的照片/视频，让桌面上的就是我家猫
5. 我可以给猫猫喂食、摸头，有互动感
6. 我可以设定每隔多久猫猫提醒我休息喝水
7. 设置简单，不需要我手动配置任何命令行或配置文件

## 功能列表

### 核心功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 桌面猫猫 | P0 | 透明悬浮窗口，常驻桌面，可拖动 |
| Claude Code 报警 | P0 | 猫猫跳动 + 气泡提示，直到用户 hover 后消失 |
| 自动配置 hook | P0 | app 自动写入/清除 Claude Code hook，用户无感 |
| 主动画上传 | P0 | 用户上传自己猫猫的图片/GIF/视频 |
| 称呼设置 | P1 | 自定义猫猫名字 + 猫猫叫你什么 |
| 休息提醒 | P1 | 每隔 N 分钟猫猫提醒喝水休息 |
| 互动（喂食/摸头/玩耍） | P1 | hover 后出现按钮，点击触发动画或粒子特效 |
| 其他素材上传 | P2 | 随机动画、互动专属动画、叫声 |

### 不在 v1 范围内

- Windows 支持
- 多只猫猫配置
- 素材云端备份
- App Store 上架

## 验收标准

**桌面猫猫**
- [ ] app 启动后猫猫出现在桌面，始终在其他窗口上方
- [ ] 透明区域可以正常点击穿透到下方窗口
- [ ] 拖动猫猫后位置保持，重启 app 后位置复原

**Claude Code 报警**
- [ ] Claude Code 触发 notification hook 后 1 秒内猫猫开始跳动
- [ ] 毛玻璃气泡出现，包含用户设定的称呼（如"主人，Claude 需要你了喵~"）
- [ ] 气泡在用户 hover 猫猫前持续显示，不自动消失
- [ ] 如果上传了叫声，同时播放一次
- [ ] app 未运行时 Claude Code 不报错（hook 静默失败）

**自动配置 hook**
- [ ] app 首次启动自动写入 hook，不破坏 `~/.claude/settings.json` 中的其他配置
- [ ] app 退出时自动清除 hook
- [ ] 设置面板显示 hook 当前状态，可手动安装/卸载

**素材上传**
- [ ] 支持上传 GIF、MP4、PNG 作为主动画
- [ ] 上传后立即生效，猫猫显示新素材
- [ ] 未上传时显示默认占位 emoji

**称呼设置**
- [ ] 设置猫猫名字和用户称呼后，报警气泡文字随之变化
- [ ] 默认值：猫猫="猫猫"，用户="主人"

**休息提醒**
- [ ] 可设置间隔分钟数（默认 45 分钟）
- [ ] 可开关
- [ ] 到点后猫猫用同款气泡提醒，文字包含用户称呼

**互动**
- [ ] hover 猫猫后出现玩耍、喂食、摸头三个按钮
- [ ] 点击后触发对应动画（有上传）或粒子特效（无上传）

## 成功指标

- 用户使用 Claude Code 时不再因为没看到等待提示而浪费超过 5 分钟
- 用户能在 5 分钟内完成首次设置（上传猫猫、设好称呼）
